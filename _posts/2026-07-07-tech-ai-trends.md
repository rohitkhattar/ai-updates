## 9. A new memory substrate tries to give AI agents actual continuity, not just a bigger context window
A new technical report out of NxtLab Innovations, "MRMS: A Multi-Resolution Memory Substrate for Long-Lived AI Agents," argues that agents which need to persist across days or weeks can't just rely on stuffing more tokens into the prompt window — they need to selectively preserve prior experience, retrieve it on demand, distinguish the agent's own reasoning from externally sourced evidence, and actively revise memories when the underlying situation changes. The multi-resolution framing means the system keeps memory at different granularities simultaneously — fine-grained recent detail alongside coarser long-term summaries — rather than the single flat store most agent-memory systems use today. It lands amid a wave of concurrent memory research this week, including papers on provenance-typed memory to stop agents confusing their own past decisions with unverified external input, and on securing long-term agent memory against poisoning attacks. For anyone building agents meant to run for more than a single session, this is a useful snapshot of where the field is converging: memory isn't one problem, it's at least four (retention, retrieval, provenance, revision), and most production agent frameworks today only solve the first.
