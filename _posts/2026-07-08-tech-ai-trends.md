## 5. DeepSeek sets a hard July 24 cutoff for its old API model names
DeepSeek confirmed today that deepseek-chat and deepseek-reasoner — the model identifiers countless integrations have hardcoded since 2025 — will be fully retired and return errors after July 24, 2026 at 15:59 UTC, with no announced extension. The fix is mechanically simple: swap the model parameter to deepseek-v4-pro or deepseek-v4-flash on the same base URL and API key, and V4 is reportedly among the cheapest frontier-adjacent models on the market. The catch is a capability trap — deepseek-reasoner maps to deepseek-v4-flash, not the more capable v4-pro, so any team that relied on deepseek-reasoner for heavy reasoning workloads and assumes the alias preserves parity will silently land on flash-tier reasoning at flash-tier prices. With Chinese open-weight models now pulling 30-46% of enterprise API token share on platforms like OpenRouter on cost alone, a botched migration that quietly degrades output quality is exactly the kind of unforced error that could reverse that momentum for teams that don't read the fine print. It's a two-week runway, not a two-month one, so this is worth checking against any hardcoded model strings today rather than in week three. For anyone running production traffic through DeepSeek's hosted API, this is a drop-everything item, not a someday one.

For anyone tracking the AI compute supply chain, it's another data point that the Nvidia-Huawei duopoly in China is facing pressure not just from geopolitics but from its own customers trying to route around both.
